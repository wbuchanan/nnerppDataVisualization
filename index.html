What advice do I have for visualizing data?

  • Mapping between visual and measurement scales should be one and the same.
  • Do not alienate potential end-users.
  • Avoid obfuscating the message.
  • Establish clear and consistent standards for visual encodings.
  • Bind helpful information in the UI without blocking the user's view.

Mapping between visual and measurement scales should be one and the same

  • Anyone have any ideas about what I might mean by this statement?
  • Why should you care about alignment of measurement/visual scales?
  • Mention that visual scales and measurement scales sometimes have different names.
  • You're going to see what will seem to be unconventional practices, but that is because a lot of these principles get thrown to the wayside for the sake of comfort and familiarity.

Nominal Scale

  • Used for categories that have no natural order.
  • Can only test equality and inequality.
  • Ask if anyone can think of any examples.
  • Ex: School Names, Demographic groups, Subject areas, etc...
  • Defined by three axoms
  • Either a == b or a != b (testing equality/inequality)
  • If a == b then b == a (defines set equivalence in two categories)
  • If a == b and b == c then a == c (extends set equivalence to multiple categories)

Ordinal Scales

  • Used for categories that have a natural order.
  • Can test for equality/inequality.
  • Can test order of magnitude.
  • No other mathematical operators are defined on this scale.
  • Ask if anyone can think of any examples.
  • Ex: Strongly Disagree to Strongly Agree, Rankings, Letter Grades, etc...
  • In addition to nominal scale properties, two additional axoms are added
  • If a > b then b !>= a
  • If a > b and b > c then a > c (defines ordering of values)

Intervallic Scales

  • Used in cases where there is no true zero or where the location of zero is arbitrary.
  • Can test for equality/inequality.
  • Can test order of magnitude.
  • Mathematical operations are defined on this scale.
  • Ask if anyone can think of any examples.
  • Ex: Z scores, Weight of humans, scaled scores (derived from Rasch), etc....
  • Builds on the ordinal scale and adds three additional axoms
  • a + b == b + a (Commutative property)
  • If a == c & b == d then a + b == c + d (Property of Equality)
  • (a + b) + c == a + (b + c) (Associative Property)

Ratio Scales

  • Used when there is a true zero or when the location of zero is fixed.
  • Has all of the same prop
  • Ask if anyone can think of any examples.
  • Ex: % of budget remaining, # of sections offered, Raw Scores
  • Adds the last two axoms
  • a + 0 == a (defines true zero value as the absence of the quantity)
  • If a == c and b > 0 then a + b > c (defines the location of 0 to be fixed)

Avoid obfuscating the message

Establish clear and consistent standards for visual encodings

  • Do you have standardized visual mappings for specific measurement scale types and/or specific data?
  • What elements do you require to be present in visualizations?
  • Are reversed or dual axes allowed in your visualizations?
  • How do you select the most appropriate type of graph to use?

Visual Scale Selection

  • You can get by with a single qualitative visual scale.
  • Leave the stoplight color palette back in the 20th century.
  • If you are going to visualize specific data in multiple locations, try to assign a consistent visual scale to the same data.
  • Since nominal scales convey exchangeable groups, you can reinforce this subtly by using the same visual encoding for all nominal scales used.
  • This also can have the added benefit of training your end users to more actively seek out and use the legend in the graph to check their assumptions/understanding.
  • This also requires communication and reinforcement of the communication
  • Use the same sequential color scale for proficiency levels to give those data a visual
